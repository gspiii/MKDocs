# Troubleshooting Issues in Outlook: Help Desk vs. System Administrator

Understanding whether an issue with Microsoft Outlook should be addressed by Help Desk support or escalated to a System Administrator is crucial for efficient problem resolution. Below is a guideline differentiating the responsibilities.

## Help Desk Support for Outlook

Help Desk typically handles straightforward, user-centric issues with Outlook. Their responsibilities include:

### 1. **Client Configuration and Setup:**
   - Assisting in initial Outlook setup and email account configuration.
   - Helping users with personal settings within Outlook.

### 2. **Basic Troubleshooting:**
   - Addressing common error messages or functionality issues in Outlook.
   - Resolving connectivity problems with the Outlook client.
   - Assisting with issues in sending or receiving emails.
   - Assisting with mailbox and calendar delegation at the user level.

### 3. **User Operation Guidance:**
   - Explaining Outlook features (e.g., calendar, tasks, rules).
   - Advising on email management and organization.

### 4. **Password Resets and Account Lockouts:**
   - Helping users with password resets and unlocking email accounts.

### 5. **Local Issues:**
   - Troubleshooting Outlook client software issues, like problems with PST or OST files.
   - Addressing performance issues (e.g., Outlook crashing or running slowly).

## System Administrator Troubleshooting for Outlook

System Administrators handle more complex, technical issues, often involving backend infrastructure. Their tasks include:

### 1. **Server-Side Issues:**
   - Resolving problems on the Exchange server affecting email delivery or server connectivity.
   - Managing server resources and configurations impacting Outlook.

### 2. **Network and Connectivity Issues:**
   - Troubleshooting network-related problems affecting Outlook's mail server connectivity.

### 3. **Security and Compliance:**
   - Handling email security policies and settings impacting Outlook.
   - Configuring encryption, archiving, and data retention policies.

### 4. **Large Scale Deployments:**
- Managing deployment of Outlook updates or upgrades.

> **Note:** Outlook update deployments are currently managed by the Desktop Engineering team, not RCIT.

### 5. **Advanced Configuration:**
   - Solving complex configurations like group policies or automated scripts.
   - Troubleshooting integration issues between Outlook and server-based applications.
